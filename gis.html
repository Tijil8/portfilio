stage 01 · ingest

Cataloguing satellite imagery with STAC

Earth-observation imagery arrives as enormous scenes tagged with time, footprint and band metadata. I worked with STAC (SpatioTemporal Asset Catalog) to index those scenes so they can be searched by area of interest and time window instead of by hand. 

# query a STAC catalog for scenes over a region + date range
items = catalog.search(
    intersects = region_geometry,   # area of interest
    datetime   = "2025-01-01/2025-06-30",
    query      = { "eo:cloud_cover": {"lt": 20} },
).item_collection()
  • Filtering scenes by footprint, date, and cloud cover before anything heavy runs
  • Validating catalog availability across many regions in batch
  • Turning region identifiers into geometries the pipeline can actually query on
stage 02 · process

Batch geospatial pipelines

Once the right scenes are found, they're processed region-by-region: reprojected, clipped to boundaries, and reduced to the values that actually matter downstream — run as repeatable batch jobs rather than one-off scripts.

region list→ resolve geometries→ STAC search→ clip + reduce→ write to PostGIS
  • Raster ↔ vector operations with the Python geospatial stack (rasterio, shapely, GeoPandas)
  • Idempotent batch runs over large region lists, resumable when they stop midway
  • Data-quality checks so bad or missing scenes never silently corrupt an output
stage 03 · serve

APIs over a spatial database

Processed results land in PostgreSQL / PostGIS, exposed through Python service APIs so front-ends can ask spatial questions — "what's inside this boundary?", "what changed over time?" — without touching raw imagery.

  • Spatial indexes and PostGIS queries for fast area/point lookups
  • REST APIs returning GeoJSON that maps can render directly
  • Deployed and run on AWS
stage 04 · visualise

React dashboards on top of the data

  • Interactive map front-ends in React, rendering the geospatial layers the APIs serve
  • Filtering and drill-down by region and time so users explore, not just view
  • The last mile — making heavy geospatial data feel light and usable
