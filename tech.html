system 01 · combat

A deterministic parry state machine

Combat is a finite state machine with frame-accurate timing windows. The enemy telegraphs, a parry window opens for a fixed number of milliseconds, and the resolution is fully deterministic — the same inputs always produce the same outcome, which makes the timing learnable, not luck. 

// each attack cycles through timed phases
enum class Phase { PlayerMenu, EnemyWindup, ParryWindow, Resolve };

// a perfect parry lands inside the inner window → counter + no damage
if (input == PARRY && elapsed <= windowMs)
    resolveParry(elapsed <= perfectMs ? PERFECT : GOOD);
  • Per-enemy tuning: windup, parry window, and perfect window in milliseconds
  • A zero-damage training dummy teaches the rhythm before any real fight
  • Difficulty scales the windows — tighter timing, faster tells
system 02 · emergent ai

Boids — flocking from three rules

The ambient life in the world isn't scripted. It's a classic boids simulation: each agent follows only three local rules, and coordinated flocking emerges — no leader, no central control. 

// three local rules, applied per-agent, every frame
velocity += separation(neighbors)   // don't crowd
          + alignment(neighbors)    // match heading
          + cohesion(neighbors);    // steer to center
  • Neighbour queries per frame, tuned for stable 60fps with many agents
  • Emergent behaviour — the flock looks intelligent with zero scripting
system 03 · rendering & world

Tile atlases, a following camera, and per-act worlds

  • Custom tile-atlas renderer with point-filtered pixel art, drawn from packed sprite sheets
  • A Camera2D that follows the player and clamps to each room's bounds
  • Hand-authored maps per act, each with its own layout, props, NPCs and interaction zones
  • A world/entity model: walkers, signposts, pickups and trigger zones as data, not hard-code
system 04 · story engine

A cutscene & dialogue state machine

The narrative runs on its own event-flag system — one integer gates progression, another carries momentum forward. Scripted sequences feed a dialogue box, then hand control back to exploration. Classic RPG architecture, built from first principles.

explore→ trigger zone→ cutscene→ fight→ resolve + carry momentum
system 05 · the build

C++ source → WebAssembly, no plugins

The exact same C++ source compiles two ways: a native binary for fast local testing (with a headless screenshot harness and a self-test suite), and a WebAssembly bundle for the web — no browser plugins, no downloads.

C++17 source→ emscripten→ .wasm + .js→ runs in any browser
  • Headless render harness for automated screenshot verification
  • A self-test suite asserting combat math (perfect-parry = 0 damage, etc.)
